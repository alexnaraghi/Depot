<task type="auto">
  <name>Task 1: Blue-tinted dark theme CSS variables and font stack</name>
  <files>src/index.css</files>
  <action>
Update the `.dark` CSS variable block in `src/index.css` to use blue-tinted HSL values inspired by VS Code's dark theme. The key change: shift hue from `0` (neutral gray) to `220` (blue-gray) for background/surface colors, and set the accent to VS Code blue (`217 91% 60%` which is #007acc).

Specific variable updates for `.dark`:
- `--background`: `220 13% 10%` (dark blue-gray, like VS Code's editor background)
- `--foreground`: `220 9% 93%` (cool white text)
- `--card`: `220 13% 10%` (same as background)
- `--card-foreground`: `220 9% 93%`
- `--popover`: `220 13% 13%` (slightly lighter for popovers/dropdowns)
- `--popover-foreground`: `220 9% 93%`
- `--primary`: `217 91% 60%` (VS Code blue accent)
- `--primary-foreground`: `0 0% 100%` (white on blue)
- `--secondary`: `220 13% 18%` (panel backgrounds)
- `--secondary-foreground`: `220 9% 93%`
- `--muted`: `220 13% 18%` (muted surfaces)
- `--muted-foreground`: `220 9% 55%` (muted text, readable but subdued)
- `--accent`: `220 13% 18%` (hover backgrounds)
- `--accent-foreground`: `220 9% 93%`
- `--destructive`: `0 62.8% 30.6%` (keep red for destructive)
- `--destructive-foreground`: `0 0% 98%`
- `--border`: `220 13% 20%` (blue-tinted borders)
- `--input`: `220 13% 20%` (input borders)
- `--ring`: `217 91% 60%` (focus ring = VS Code blue)

Also add a `body` font-family in the base layer:
```css
body {
  @apply bg-background text-foreground;
  font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
}
```

Leave `:root` (light mode) variables unchanged — dark only app but keep them for Radix defaults.
  </action>
  <verify>Run `npx tsc --noEmit` to ensure no type errors. Visually: open the app and confirm blue-tinted dark background instead of pure gray.</verify>
  <done>CSS variables use blue-tinted HSL values with hue ~220. Font is system font stack. App renders with blue-gray dark theme.</done>
</task>
